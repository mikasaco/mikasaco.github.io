List需要知道的事儿

发表于 2019-09-17 | 分类于 集合 ， List ， ArrayList ， LinkedList ， Vector

摘要

List是有序的，存入的顺序和取出的顺序一样。

  1. ArrayList

    初始化方式 说明
    ArrayList() 无参构造器，刚初始化出来是一个空数组DEFAULTCAPACITY_EMPTY_ELEMENTDATA，会在第一次add的时候扩容成10。
    ArrayList(0) 初始化出来的数组是EMPTY_ELEMENTDATA，是一个空数组，在第一次add的时候扩容成1。
    ArrayList(10) 初始化出来的数组长度是10，相比于ArrayList()会少了一次扩容的过程。

    时间复杂度

    操作 时间复杂度
    get() 根据下标查询，时间复杂度O(1)，ArrayList只有这种这一种get操作
    add(E) 在尾部添加元素，时间复杂度O(1)
    add(index,E) 指定位置添加元素，需要移动元素，时间复杂度O(n)
    remove(E) 删除指定元素，需要移动元素，时间复杂度O(n)

  2. LinkedList

    时间复杂度

    操作 时间复杂度
    get() getFirst和getLast直接取头部和尾部的元素，时间复杂度O(1)，get(index)需要遍历，时间复杂度O(n)
    add(E) 默认是添加在尾部，时间复杂度O(1)
    add(index,E) 指定位置添加元素，需要遍历，时间复杂度O(n)
    remove() 默认是删除头部元素，时间复杂度O(1)
    remove(index,E) 指定位置删除元素，需要遍历，时间复杂度O(n)

前言

List应该是日常开发中最常用的数据容器，List是有序集合，ArrayList、LinkedList和Vector也是面试中的必考题，透彻研究List中的主要接口和类是必须的。本文会对List的主要接口和ArrayList、LinkedList、Vector的主要源码进行，分析差异和数据结构，主要分为以下几点：

  1. 数组和链表的数据结构简述
  2. ArrayList主要源码分析
  3. LinkedList主要源码分析
  4. Vector主要源码分析
  5. ArrayList、LinkedList、Vector差异对比

数组和链表

数组和链表是基本的数据结构，都是线性表的一种，List的数据结构就是以这两个为基础的，这里简单过一下。TODO 写个数据结构博客

数组是在内存中连续的一块区域，因此其特点是：

顺序存储的线性表。时间复杂度：索引取值O(1)；据值查找O(n)；插入O(n)；删除O(n)。

链表有单链表、双向链表、循环链表，实现是在每个元素中额外增加指针，指向下一个（双向链表还会指向上一个）元素，其特点：

链式存储的线性表。时间复杂度：索引取值O(n)；据值查找O(n)；插入O(1)；删除O(1)。

ArrayList主要源码分析

ArrayList底层实现是数组，也就是内存中连续的一块区域，跟数组不同的是，ArrayList是可以动态扩展的。

看源码不需要具体到所有的源码内容，这样看要累死，而且要看晕，抓不住重点。对于ArrayList我们需要掌握的主要是下面这几个部分：

  1. 初始化
  2. 扩容
  3. 删除
  4. 时间复杂度

初始化

ArrayList中有三种构造方式，如下图。

我们先看下传入参数为int的构造器。代码很简单，根据传入的值去创建数组（elementData就是实际存放的数据数组），而如果传入的值为0，那么让elementData = EMPTY_ELEMENTDATA，也就是长度为0的空数组。关于EMPTY_ELEMENTDATA 后文会和DEFAULTCAPACITY_EMPTY_ELEMENTDATA做对比。

我们再看一下传入参数为Collection的构造器，其实现是先讲Collection集合对象转为数组elementData，再判断数组size是否为0，如果为0就让elementData = EMPTY_ELEMENTDATA。

最后再看下无参构造器。代码就一行，让elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA，这里的DEFAULTCAPACITY_EMPTY_ELEMENTDATA和EMPTY_ELEMENTDATA都是长度为0的空数组，这两者有什么区别呢？

DEFAULTCAPACITY_EMPTY_ELEMENTDATA和EMPTY_ELEMENTDATA区别

我们先看一下这两个空数组的定义。需要注意这两个空数组都是static修饰的，会在方法区中存放，这么做是为了缓存。

简单来说，EMPTY_ELEMENTDATA就是一个共享的空数组，初始化ArrayList时如果传入参数size为0，就用他，如果是无参构造器就用DEFAULTCAPACITY_EMPTY_ELEMENTDATA；DEFAULTCAPACITY_EMPTY_ELEMENTDATA区别于EMPTY_ELEMENTDATA的地方在于，两者扩容时的操作不一样，因此需要区分。那到底怎么不一样呢，我们举个例子。

当参数为0时，创建的数组elementData = EMPTY_ELEMENTDATA，此时如果add一个元素，之后的数组size为1。

当用无参构造器时，创建的数组elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA，此时如果add一个元素，之后的数组size为10。

这是为什么呢？原因在这扩容时调用的这段代码中。

new ArrayList()和new ArrayList(10)的区别

经常会看到ArrayList的默认大小是10，new ArrayList()和new ArrayList(10)有什么区别呢？

那看了上述分析，这个问题很好解答，new ArrayList()其实初始化的是空数组，会在add第一个元素的时候，扩容成10，相比于new ArrayList(10)，多了一次扩容。

扩容

ArrayList的扩容已经是个老生常谈的话题的，网上资料一大堆就不具体源码分析了，这里阐述一下大致的流程：

真正的扩容方法在grow()中，扩容是原长度的1.5倍。

ArrayList的add有两种：默认的add()方法是添加在尾部的，还有一个指定位置的add(index,e)，后者需要移动元素，跟remove类似。

remove

前文也说过，ArrayList是基于数组实现的，当删除元素时需要移动后面的元素，ArrayList的remove方法的核心：通过System.arraycopy拷贝的方法，给删除元素index后面的元素复制。

System.arraycopy拷贝性能高，很多源码都是用这个来拷贝数组

时间复杂度

ArrayList基于数组，查询快，增删慢。

操作 时间复杂度
get() 根据下标查询，时间复杂度O(1)，ArrayList只有这种这一种get操作
add(E) 在尾部添加元素，时间复杂度O(1)
add(index,E) 指定位置添加元素，需要移动元素，时间复杂度O(n)
remove(E) 删除指定元素，需要移动元素，时间复杂度O(n)

时间复杂度怎么看呢？

举个例子，ArrayList remove时，如果元素在末尾，不需要移动，时间复杂度O(1)，而如果remove的是第一个元素，需要移动后面的每个元素，时间复杂度O(n)，我们算时间复杂度需要算的是最坏的情况，所以是O(n)。

LinkedList主要源码分析

对于LinkedList我们首先需要知道的是，LinkedList是双向链表，其内部定义了一个静态类。

对于LinkedList，我们源码需要掌握的有：

  1. add/remove
  2. get
  3. 时间复杂度

add/remove

LinkedList是双向链表，我们add/remove元素有3种方式：尾部、头部、中间任意位置。

LinkedList默认的add()方法就是添加在尾部的

删除元素remove(0默认是从头部开始删除的

add/remove的工作原理都很清楚，add就是new一个node，然后让上一个元素指向new node，remove就是让上一个元素指向下一个元素。然后需要知道的一个是，在中间位置add/remove的时候需要找到index的位置。无论是add还是remove 中间位置元素时，

  1. 校验索引大小 0<=index<=size
  2. add方法会判断是否是尾部，是的话直接在尾部插入了。
  3. 遍历查找index的位置，在add/remove元素。

这里查找的node()方法，会判断索引index和size/2的大小，然后再遍历找到添加的位置，这样做提高了效率，不用每次都从头或尾开始遍历。

get

都知道链表的查询是O(n)的时间复杂度，需要遍历链表才能获取，LinkedList也差不多，但由于是双向链表，遍历的node()方法会判断索引index和size/2的大小，然后再遍历找到添加的位置，提高了效率。

LinkedList有3种get()方法：getFirst(),get(),getLast()。get()方法会调用node()方法，遍历查找。

时间复杂度

LinkedList是基于双向链表，查询慢(需要遍历)，增删快。

操作 时间复杂度
get() getFirst和getLast直接取头部和尾部的元素，时间复杂度O(1)，get(index)需要遍历，时间复杂度O(n)
add(E) 默认是添加在尾部，时间复杂度O(1)
add(index,E) 指定位置添加元素，需要遍历，时间复杂度O(n)
remove() 默认是删除头部元素，时间复杂度O(1)
remove(index,E) 指定位置删除元素，需要遍历，时间复杂度O(n)

vector主要源码分析

vector也是基于数组，跟ArrayList类似，我们主要掌握的有三点：

  1. 线程安全
  2. 扩容
  3. 初始化

线程安全

vector的线程安全是靠synchronized关键字实现的，在方法上加锁。

扩容

跟ArrayList类似，最终的扩容也是在grow方法中。

那扩容多少呢？关键看这行代码。

int newCapacity = oldCapacity + ((capacityIncrement > 0) ? capacityIncrement : oldCapacity);

capacityIncrement这个是Vector的增长因子，可以在初始化的时候指定。这行代码的意思是，如果指定了增长因子，那么扩容的时候是原始大小+增长因子；如果没有指定(默认)是两倍。
