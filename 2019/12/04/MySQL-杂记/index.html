发表于 2019-12-04| 更新于 2019-12-04|MySQL杂记

前言

本文记录的内容比较分散，都是MySQL中些零散的内容。

Buffer Pool

作用

Buffer Pool对数据更新/查询性能都有很大的影响，简单来说就是磁盘数据缓存在内存了，可以减少IO。

  • 更新数据的时候，事务执行完了不会立刻写到磁盘，而是记录在Buffer Pool(change buffer最终也是要映射到 buffer pool)，然后在数据刷盘时机到了的时候再写到磁盘。
  • 查询数据的时候，如果 buffer pool 中有这个数据页的数据，就不用再去磁盘查找了，直接内存中返回了。

独特的LRU算法

Buffer Pool是链表形式的数据结构，也是用的LRU淘汰算法来实现数据缓存。但和一般的LRU淘汰算法不一样的是，buffer pool的LRU是分段的，分了old区和young区，靠近链表头部的5/8是young，靠近链表尾部的3/8是old。

  • 当我们查询一个数据页时，如果数据页在 young 区，那放到头部；
  • 当我们查询的数据页在 old 区，需要判断这个数据页在链表中的时间，如果小于1秒，不动；如果大于1秒，移到头部；
  • 当我们查询的数据页不在buffer pool，那么从磁盘取过来之后会放到old区。

这种设计是为了防止当我们一个查询全表扫描某张表，会使得所有原来的链表中的数据页都被淘汰了，保证了不会响应正常业务的查询命中率。

结果集

初始MySQL结果集，总是会认为是内存中一块区域，然后会把结果拼凑好之后，一次返回给客户端。但这种认识有正确，也有问题的地方。

结果会先存在 net_buffer ，这个 net_buffer 默认大小16K，存满了就发送给客户端。所以确实是有一块区域来存放结果，但MySQL是边读边发，不是一次把所有结果返回的。
