Dubbo-服务引用
发表于 2019-12-29| 更新于 2020-01-15|Dubbo服务引用

前言

使用 Dubbo，只需要服务消费者引用服务提供者的接口jar包(泛化调用可以不用服务提供者的接口jar包)，就可以像调用本地方法一样，调用远程的服务。本文就分析一下服务引用的原理。

这里所说的服务引用是 consumer 配置解析以及创建代理类的过程，而不是一次服务调用的过程。

服务引用流程

  1. 根据配置文件 <dubbo:reference init=”true”> init的配置走不同的服务引用时机，最终都走到同样的路口:ReferenceBean#getObject；

  2. 然后会进行一系列的参数封装，主要是拼接URL；

  3. 根据URL协议的不同会有3种不同的服务引用方式：

    a. 协议是 injvm，走本地虚拟机的服务引用；

    b. 如果有配置<dubbo:reference url>直连引用，这时候的URL例如 dubbo://service-host/com.foo.FooService?version=1.0.0，根据协议 dubbo 会直接走 DubboProtocol；

    c. 协议是 registory，走注册中心的服务引用；

  4. 下面开始的分析都是走注册中心的服务引用方式。协议头是 registory 根据 SPI-Adaptive，会调用 RegistryProtocol ，然后又会根据 URL 中参数 protocol 走具体的注册中心(ZookeeperRegistry)，接着连接注册中心、注册consumer节点、订阅provider、router、configurators节点；

  5. 然后根据具体引用的协议(默认 dubbo 协议)会去引用服务，引用服务做的事情就是创建客户端，默认是 Netty，并且连接远程服务提供者，如果设置 lazy 参数为 true，那么会在真实服务调用的时候才去连接；

  6. 对于多个服务提供者或者多个协议等情况，最后会用 Cluster 接口把所有的 invoker 都封装成一个统一的 invoker 方便调用，这样就完成了远程接口转换成 invoker 对象的过程；

  7. 对 invoker 对象生成代理类，默认是用 JavassistProxyFactory 代理工厂生成的，会拼接字符串最终调用 toClass() 动态生成一个代理类，这样就完成了 invoker 对象到代理类的过程。

服务引用的时机

服务引用使用 < dubbo:reference> 标签，按照是否配置 init=”true” 会有两种服务引用的时机：

  • 配置了 <dubbo:reference init=”true”> 会在解析到这个标签的时候就进行服务引用，也就是饿汉式的方式。
  • 配置了 <dubbo:reference > 并不会在解析这个标签的时候进行服务引用，而是在获取这个 bean 的时候进行服务引用，也就是懒汉式的方式。

饿汉式的服务引用原理是 ReferenceBean 标签实体类实现了 InitializingBean 接口，会在初始化 Bean 的时候就执行 afterPropertiesSet 方法，并且 afterPropertiesSet 方法会判断 init =true 是否成立，成立的话就调用 getObject 方法，getObject 方法是服务引用的入口。

而懒汉式的服务引用原理是 ReferenceBean 实现了 FactoryBean 接口，这个接口会在获取 Bean 的时候调用 getObject 方法。对比这两种服务引用的时机，最后都是调用了 FactoryBean 接口的 getObject 方法，唯一的不同就是在初始化 Bean 的时候执行 afterPropertiesSet 方法中会判断 init 是否为 true。

获取注册中心信息

我们主要分析最常用的使用注册中心的服务引用方式。本节就分析一下是怎么获取注册中心的。其最根本的还是获取到的我们 spring.xml 中配置的注册中心信息。

  1. DubboNamespaceHandler 会先解析 < dubbo:registry> 标签，然后创建一个 RegistryConfig 实体类；
  2. < dubbo:registry>标签解析初始化 Bean 的时候调用 afterPropertiesSet 方法，会判断 registries 是否为空，如果为空的话，会使用 BeanFactroyUtils 获取容器中之前解析的 RegistryConfig；
  3. 然后在调用 loadRegistries 方法获取注册中心(服务暴露和服务引用都会调用这个方法获取注册中心)，所以就能获取到我们 spring.xml 配置的 < dubbo:registry> 的注册中心信息。
