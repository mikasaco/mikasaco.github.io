发表于 2019-12-05| 更新于 2019-12-11|源码解读jdbc

前言

众所周知，JDK提供了JDBC的实现规范，而具体的实现由第三方厂家提供，所以可以接入MySQL、Orcale等不同厂家的数据库，这种提供接口，而让第三方自己实现的方式就是SPI，JDK的SPI实现是一种标准的策略模式实现。

SPI的使用方式

我们先不看源码，自己使用一下JDK提供的SPI功能。我们定义一个 SpiDemo 接口，和其实现 MySpi 。在代码中我们创建 MySpi ，但并不直接指定 MySpi ，看如下代码。

在这个文件中只有一行 com.shenlinqiang.springdemo.spi.MySpi 实现类的全路径。

以上步骤，就实现了代码中不指定实现类，通过配置文件获取实现类的使用了，总结一下使用方式：

  1. 定义一个接口和实现类；
  2. 资源文件中新建目录 META-INF/Services/接口全路径，文件内容写 实现类全路径；
  3. 代码中通过 ServiceLoader 及其内部实现的迭代器初始化实现类。
