6.4 contentEquals/equals/contains

6.4.1 contains

contains是用来判断string对象中有无指定字符序列，判断的原理其实就是调用indexOf，如果indexOf的返回值为-1，那就是没有

注意是字符序列不是字符串，字符序列是指实现了CharSequence接口的类，包括String，StringBuffer，StringBuilder；字符串就是String

6.4.2 contentEquals

contentEquals是判断两个字符序列是否相等，传入参数可以是String，StringBuffer，StringBuilder，如果是String直接调用equals方法了，如果是别的字符序列就逐个对比

6.4.3 equals

这个equals方法是重写了Object的，其源码的实现逻辑很简单，就是逐个去比较两个String对象的char数组，如果都相等返回true，有不相等的，或长度不一就返回false，其实跟上面的contentEquals差不多，只不过contentEquals比较的是字符序列，所以String可以和StringBuffer，StringBuilder等比较，而equals只能和String比较

6.5 getChars

获取String对应的char数组，乍一看好像直接可以返回value，但其实，源码这里做了一次复制，所以其实返回的是另一个char数组，这么做的原因也是为了String的安全，维护不可变

为什么选择31作为乘积因子，而且没有用一个常量来声明？主要原因有两个：

　　①、31是一个不大不小的质数，是作为 hashCode 乘子的优选质数之一。

　　②、31可以被 JVM 优化，31 * i = (i << 5) - i因为移位运算比乘法运行更快更省性能。

6.7 intern和常量池

这里先说一下String的常量池（字符串缓冲区）参考博客

Java运行时会维护一个String Pool， 也叫“字符串缓冲区”。String池用来存放运行时中产生的各种字符串，并且池中的字符串的内容不重复。对于创建String对象，有以下几种情况：

  1. 字面量创建字符串或者纯字符串（常量）拼接字符串会先在字符串池中找，看是否有相等的对象，没有的话就在字符串池创建该对象；有的话则直接用池中的引用，避免重复创建对象。

  2. new关键字创建时，直接在堆中创建一个新对象，变量所引用的都是这个新对象的地址。

  3. 使用包含变量表达式来创建String对象，则不仅会检查维护字符串池，还会在堆区创建这个对象，最后是指向堆内存的对象。

7. String的不可变

一般来说String确实是不可变的，但看了源码之后可以知道，其实数据就是在char数组中，而这个char数组基本不能在外部获取，但其实可以通过反射的方式来改变char数组，也就是改变了这个String，当然基本不会这么用

总结一下这类问题，关键是两点：
1.String常量池的存在，会让String a = "abc"这种在常量池创建常量，a持有的是对于常量池的引用;
2.对于像String b = "a"+"b"这种能够在编译期确定的（final或+两边是常量），编译器会进行优化，结果就等同String b = "ab"

另外对于对象的创建：
栈中用来存放一些原始数据类型的局部变量数据和对象的引用(String,数组.对象等等)但不存放对象内容
堆中存放使用new关键字创建的对象.
字符串是一个特殊包装类,其引用是存放在栈里的,==而对象内容必须根据创建方式不同定(常量池和堆)==.有的是编译期就已经创建好，存放在字符串常 量池中，而有的是运行时才被创建.使用new关键字，存放在堆中。

这段代码在a=b的时候会编译不通过，这是因为final的不变性是指==final修饰的变量其指向的地址不能改变==，这里a=b对a的赋值改变了a的指针地址，因此不能编译通过
==总得来说对于一个final变量，如果是基本数据类型的变量，则其数值一旦在初始化之后便不能更改（内容不可变）；如果是引用类型的变量，则在对其初始化之后便不能再让其指向另一个对象（指针不可变，内容其实是可变的）==
