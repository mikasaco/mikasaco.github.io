核心API-processEngine流程引擎

流程引擎processEngine是整个Activiti的核心，对于开发者，它是我们使用Activiti的facade，通过它可以获得所有想要的service。可以看到processEngine是一个接口，并提供了获取各种service的方法。

如何创建processEngine的？

既然知道processEngine是由配置对象去创建的，那么我们看ProcessEngineConfigurationImpl的源码，里面的buildProcessEngine方法，就是创建了一个processEngine对象，并返回给开发者使用。

之前也分析过了init方法，就是进行了一系列初始化操作，包括initServices、数据源、拦截器、视图解析等等。

总结一下流程引擎对象processEngine：

  1. processEngine是Activiti的门面接口，提供了访问各种service的方法，是使用Activiti必须用的对象。
  2. processEngine的创建是通过ProcessEngineConfigurationImpl的buildProcessEngine创建的，在new ProcessEngineImpl之前，也进行了一些初始化的操作。
