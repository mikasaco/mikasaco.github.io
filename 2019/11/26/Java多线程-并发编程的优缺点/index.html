Java多线程-并发编程的优缺点
发表于 2019-11-26| 更新于 2019-12-04|Java多线程起篇

1. 为什么要用并发编程(优点)

硬件发展中有个很著名的”摩尔定律”，意思是CPU处理速度过几年会翻倍。但随着发展CPU发展到了瓶颈，大概主频在4GHz就很难再上升了。但我们计算机还是要发展的，于是想到用多核CPU来并发处理，而并发编程就是针对于多核CPU的，能够最大化多核CPU的性能。

所以并发编程的最大优点就是 “压榨”多核CPU的处理能力。

1.1 多核CPU和多个CPU

这些计算机硬件的基础知识还是有必要弄懂的，经常会对CPU、处理器、多核CPU、多CPU给搞混了，这里梳理一下：

  1. CPU就是处理器，而核是CPU中真实存在的，是CPU的处理单元；
  2. 多核CPU(也就是多核处理器)就是CPU里面有多个核；
  3. 多CPU是计算机上有多个CPU，多个CPU可以并行执行，区别于多核CPU的并发执行。

2. 并发编程带来的问题(缺点)

并发编程可以充分利用多核CPU的性能，但也给开发带来了难度，处理不当就可能带来并发安全问题。主要带来的问题有 点：

  1. 多线程并发访问共享数据时，可能会有数据不一致的问题，甚至还有死锁情况；
  2. 线程上下文切换消耗资源

线程上下文切换的理解

Java使用抢占式的方式获取CPU资源，一个线程执行一个时间片(几十毫秒)之后又要去抢占CPU资源，而线程操作需要的数据，我们在这种上下文切换的时候需要保存和恢复；

此外Java线程是使用1对1映射内核线程的，所以线程切换需要内存态介入，也就需要用户态和内核态的切换，这也是线程切换带来的性能损耗

3. 并发编程中的一些概念

3.1 并发和并行

并发是多个线程抢占CPU资源，然后执行各自的代码指令，因为时间片很小，看上去是一起执行的，但严格时间上来说并不是一起执行的，一般就是多核CPU；

并行大多是多个CPU，多个CPU可以在同一时刻处理不同线程的代码指令。

3.2 同步和异步

同步是调用者调用了同步方法之后必须等待在那里，等待被调用者结果返回之后才可以接着执行下面的代码；

异步是调用者异步方法执行之后立刻返回，然后等到被调用者有结果了获取处理的结果就可以了。

比如买东西，去实体店买东西需要选了东西，等待商家拿给你；而网上购物只要选好付钱了，商家会快递送过来。

3.3 临界区

临界区就是多线程访问的共享数据。
