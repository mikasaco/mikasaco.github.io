Java多线程-线程的状态和基本操作
发表于 2019-11-26| 更新于 2019-12-10|Java多线程线程池

1. 如何创建线程

Java中一般3种方式创建线程：

  1. 继承Thread类(不推荐)；
  2. 实现Runnable接口；
  3. 实现Callable接口。

Runnable接口没有返回值，Callable接口有返回值。Callable使用需要调用ExecutorService.submit()方法。

3. 线程的基本操作

3.1 join

  1. 线程A调用了线程B的join()方法，那么等线程B执行完了才会再执行线程A；

  2. join是线程的方法：Thread.join()；

  3. join()方法不会让线程进入blocked状态，而是通过循环的方式等待。

3.2 sleep&wait

  1. sleep是Thread类的方法，wait是Object类的方法；
  2. sleep使用，休眠了指定时间后就线程就可以去抢占CPU资源，而wait使用后必须等待别的线程调用了notify()才可以唤醒；
  3. wait使用一般配合synchronized，需要锁，在调用了wait之后会释放锁，让别的线程去获取锁；而sleep不会释放锁，锁还是被原来的线程持有。

3.3 yield

让线程释放当前的CPU资源，然后和该线程同一个优先级的线程一起再去抢占CPU资源。
