发表于 2019-11-18| 更新于 2019-12-04|源码解读集合Set

一图概括

前言

Set也是Java集合的重要部分，在日常使用中，我们总是会利用Set集合元素不重复的特性来去重，本文对HashSet、LinkedHashSet和TreeSet这三种使用较多的Set API进行分析，Set的源码都比较简单，直接列举重点了。

Set实现元素不重复的原理

Set实现元素不重复是其最大的特点，而无论是HashSet、LinkedHashSet还是TreeSet，其实现元素不重复的原理都是相同的——借助Map实现。

那么怎么利用Map来实现元素不重复呢？

Map在put元素的时候，会根据key去查找，如果已经有了这个key(hashCode+equals)，那就更新替换value，而不插入数据。

Set正是充分利用了这个特性，将元素当成Map的key，value都是一个new Object()，这样每次add元素的时候，其实都是去校验了key是否存在。

HashSet

  1. HashSet内部维护了一个HashMap，使用HashMap的key来存储元素；
  2. HashSet允许存在元素为null的，因为HashMap允许key为null；
  3. HashSet是无序的，因为HashMap的key是无序的；
  4. HashSet没有get()方法。

LinkedHashSet

  1. LinkedHashSet继承HashSet，其构造也都是用的HashSet创建LinkedHashMap的那个构造方法；
  2. LinkedHashSet其实内部维护的是LinkedHashMap，且是插入顺序访问的模式，因此是有序的；
  3. 其他的跟HashSet都一样，包括允许元素为null，没有get()方法。

TreeSet

  1. 内部用TreeMap或NavigableMap来存储数据；
  2. 按照元素大小有序。
