什么是IOC，什么是DI

说到 Spring 肯定会提到 IOC 和 AOP，也会说 Spring 撑起了 Java 语言的一片天，那为什么 Spring 在服务端开发如此重要呢？

我们设想如果没有 Spring ，或者说没有 IOC 容器，那我们开发一个应用系统会异常复杂麻烦，最主要体现在两个方面：

  1. 需要频繁创建对象实例；
  2. 对象之间的依赖关系复杂；

Spring 是怎么解决这两个问题的呢？对于第一点“需要频繁创建对象实例”，Spring 是通过 IOC 容器来解决的；对于第二点“对象之间的依赖关系复杂”，Spring 是通过依赖注入（DI）来解决的。
