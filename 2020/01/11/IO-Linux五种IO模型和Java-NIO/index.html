IO-Linux五种IO模型和Java NIO
发表于 2020-01-11| 更新于 2020-02-13|IOIO模型

缓冲IO带来的思考

缓冲 IO （buffer IO）也称作标准 IO，是系统默认的 IO 方式，使用缓存 IO 会先将磁盘文件读入内核空间的缓冲区（也就是页缓存 page cache），然后再从内核空间缓冲区拷贝到用户空间的缓冲区。缓冲 IO 利用了 page cache 的预读功能，可以在后续的读数据的时候减少 IO 操作。

和缓冲 IO 对应的就是直接 IO（Direct IO），具体可以看之前的博客。

所以在标准 IO 中，我们的 IO 被分为了两个部分：

  • 数据准备；
  • 数据从内核空间拷贝到用户空间。

数据准备包括 用 DMA 从磁盘把文件读入内存、或者是网络传输 socket 等。

Linux的5种IO模型

针对上面标准 IO 的两个节点，Linux 有了 5 种 IO 模型：

  1. 阻塞 IO；
  2. 非阻塞 IO；
  3. 多路复用 IO；
  4. 信号驱动 IO；
  5. 异步 IO。

阻塞IO

  1. 阻塞 IO 就是数据准备和数据拷贝两个阶段，用户线程都是阻塞着的。从发起 IO 到获取到 IO 结果都是阻塞；
  2. Linux 默认的 IO 模型就是阻塞 IO。

非阻塞IO

  1. 设置 socket 参数为 non-blocking，就是非阻塞 IO 了；
  2. 非阻塞 IO 是在发起调用之后就立刻返回 IO 结果了，如果返回 error 数据没准备好；如果不是 error，就可以将数据从内核空间拷贝到用户空间，拷贝阶段是阻塞的；
  3. 一般在数据准备阶段，用户线程会不断询问数据准备是否完成；拷贝阶段就是阻塞的。

多路复用IO

  1. 数据准备阶段，用户线程调用 select() 函数之后会阻塞，如果 select() 监听的某个 socket 数据准备好了，那么 select() 函数就会返回；
  2. select() 函数返回之后，用户线程就可以继续执行，进行数据拷贝。
  3. 如果用户线程单线程（select() 数据准备和 read() 数据拷贝在一个线程），那和阻塞 IO 其实没什么区别，甚至性能更差；但多路复用 IO 的优势就在于可以用一个线程去监听多个 socket 连接，如果再把 select() 和 read() 分开就性能很好了。

一个线程监听多个 socket 连接的实现原因就是 select() 函数，这个函数会不断轮询注册在 selector 上的 socket 状态，一旦某个 socket 就绪（例如读就绪），那 select() 函数就可以返回。

select、poll、epoll

多路复用 IO 是一种 IO 模型，其核心是利用 select() 函数，用一个线程监听多个连接。

但其实 Linux 中不是就 select() 可以实现一个线程监听多个连接，另外还有 poll 和 epoll。只是 select() 函数最早，比较有代表性。

  • select() 函数有最大文件描述符（socket 就是一种文件描述符）限制，获取就绪的文件描述符时需要遍历所有注册的描述符；
  • poll() 函数没有最大文件描述符限制，获取就绪的文件描述符时需要遍历所有注册的描述符；
  • epoll() 函数没有最大文件描述符限制，采用回调的方式获取就绪的文件描述符。

因为 select() 和 poll() 获取就绪的文件描述符，需要遍历所有注册的文件描述符，在大量连接的情况下，同一时间就绪的描述符比例很少的，所以在有大量描述符的时候，性能差，epoll 更好。

信号驱动IO

用的很少。

异步IO

  1. 数据准备和数据拷贝都是异步的。

同步异步/阻塞非阻塞

阻塞/非阻塞：调用阻塞 IO，会在 IO 完成（socket 拷贝到用户空间）之前，线程都阻塞，是阻塞 IO；其他 4 种 IO 模型的都是非阻塞 IO。

同步/异步：IO 操作完成之前都不会阻塞就是异步，只要有某个阶段阻塞就是同步。所以 5 种 IO 模型中，阻塞 IO、非阻塞 IO、多路复用 IO 都是同步的；异步 IO 是非阻塞的。

按照上面定义，也把多路复用 IO 定义是同步非阻塞 IO。

Java NIO

Java 中的 IO 分为 3 种：BIO、NIO、AIO。

BIO 是最早的 IO ，对应 5 种 IO 模型就是阻塞 IO；AIO 是1.7 之后才有的，对应的是异步 IO，但这种 IO 现在还不成熟；Java 中的 NIO 不是 5 种 IO 模型中的非阻塞 IO，而是多路复用 IO。NIO 见到的最多，下面介绍一下 Java NIO 的使用和三大核心组件：channel、buffer、selector。

NIO = new IO 而不是 非阻塞 IO。

三大核心组件

channel

  1. 一个连接 = 一个 socket = 一个 channel；

  2. 与流的区别：流单向的，要么 InputStream、OutputStream，channel 是管道，数据可以双向；

  3. 常用的 channel 实现

    • FileChannel：操作文件；
    • SocketChannel：TCP 协议；
    • DatagramChannel：UDP 协议；
    • ServerSocketChannel：监听 TCP 协议的 Accept 事件，之后会创建 SocketChannel。

  4. channel 都和 buffer 一起使用，从 buffer 读出，或读到 buffer。

buffer

  1. buffer 本质上就是一个内存块，JDK buffer 使用起来比较麻烦的，读写需要切换调用 flip() 方法；
  2. buffer 常用的有：ByteBuffer、CharBuffer、MappedByteBuffer 等。MappedByteBuffer是内存映射，使用的是堆外内存。

selector

selector 就是多路复用器，重要的方法有以下三个：

  • channel.register(Selector,event)。这个是 channel 的注册方法，是把要监听的 channel 的某个事件注册到 selector 上；
  • selector.select()。这个方法底层就是去执行 OS 的 select() 等函数，实现了一个线程监听多个连接。执行了这个方法线程就阻塞了，然后如果某个连接数据准备好了，那程序就继续往下走；
  • selector.selectedKeys()。这个方法就可以获得所有就绪的 channel。selectedKeys 里面存的是channel 和 selector 。

可以看到多路复用 IO 的核心就是用 selector.select() 调用了系统的 select() 函数，实现了监听多个连接的功能。但是如果只有这个，如果单线程中后面我们处理写事件比较耗时的话，性能也是不好的。因此才有了反应器模式的 多路复用 IO，也就是用一个线程做多路复用器，调用 select() 函数，然后当接收到一批就绪的 channel 的时候，用另外的线程去处理对应的事件，这样就实现了 select() 和事件（read、write、accept 等）的分离。
