JVM-javac和JIT编译器
发表于 2020-01-28| 更新于 2020-02-01|JVM编译器

前言

Java 有三种编译器：前端编译、即时编译、AOT。本文主要介绍前两者。

JVM 执行的两种方式：

  1. Java 文件通过前端编译器(Javac)编译成 class 文件，然后虚拟机通过解释器和程序计数器配合解释执行；
  2. 对于热点代码等，会在程序运行期间通过即时编译(JIT)的方式编译成 本地代码，然后通过寄存器编译执行。

前端编译—Javac

从 Java 文件到 class 文件的过程就是编译，一般都是用 javac 来编译的，也被称作前端编译器。

Java文件到class文件的过程

从 Java 文件到 class 文件一般会经过以下这几个步骤：

  1. Java 文件；
  2. 词法分析；
  3. 语法分析；
  4. 抽象语法树；
  5. class 文件。

语法糖

Javac 编译器除了完成从 Java 文件到 class 文件的编译，性能上的优化几乎没有，但提供了编程方式上的优化—语法糖。Java 中的语法糖包括：

  1. 拆箱装箱；
  2. 泛型擦除；
  3. 循环遍历。

即时编译—JIT

Java 一般是用解释器去解释执行方法区中的基于栈操作的指令，然后操作虚拟机栈中的操作数栈，但这种基于栈操作的方式性能比基于寄存器要差，所以 Java 还提供了在运行期间，针对热点代码编译成本地代码，这就是即时编译，一般有两个层次的编译器 C1 和 C2。

关于即时编译，我们需要掌握下面这些问题。

JVM为什么是编译器和解释器并存的？

  1. 编译器和解释器各有优劣，编译器把 class 文件编译成本地代码，执行性能高了，但编译需要花费时间，启动就慢了；解释器虽然执行性能差了点，但可以省去编译的时间，启动快。JVM 这种并存的设计可以让程序启动更快，并且在启动之后对于热点代码再编译，是兼顾启动速度和执行性能的方案。
  2. 如果采用激进的编译方式，解释器还可以作为逃生门，回退到解释器执行的方式，让程序得以执行下去。

Hotspot为什么有两个不同的即时编译器？

Hotspot 中有两种即时编译器：Client 编译器和 Server 编译器，这两种编译器编译出来的本地代码质量不一样，Server 编译器编译的性能更好，但需要更长的编译时间，正是由于这种性能和编译时间的反比关系，所以才需要有两个不同的即时编译器。

在 Hotspot 中采用分层的编译策略：

  • 0层。只用解释器；
  • 1层。用解释器和 Client 编译器，编译速度较快，执行性能较低；
  • 2层。用解释器和 Server 编译器，编译速度较慢，执行性能较高。

何时触发即时编译？

对于热点代码会触发即时编译，热点代码包括两类：

  1. 多次执行的方法；
  2. 多次执行的循环体。这种也称为帧上替换，因为程序还在帧上运行就替换了。

而如何判断代码被多次执行，也有两种方式：

  1. 采样的热点探测；
  2. 计数的热点探测(Hotspot 采用的是这种)。主要就是统计方法调用次数和回边次数(为了统计循环体)。

编译优化方式有哪些？

方法内联

A 方法中调用 B 方法，方法内联之后 A 不调用 B 了，而是直接在 A 方法中运行 B 方法的内容。

好处

去掉了方法调用的成本，如建立栈帧。

公共子表达式消除

如果一个表达式 E 已经计算过了，并且在再次出现之前没有改变过，那么再次出现的时候可以不计算，直接用就可以。

好处

减少计算了，有时也减少了堆中对象数据的访问。

数组边界检查消除

Java 需要判断数组是否越界，如果这种判断很多也是浪费性能的，所以如果可以确定不会越界，那么就可以优化，消除检查。

逃逸分析

逃逸分析是分析对象的作用域，有两种：

  1. 如果对象定义后可能被别的方法访问，例如作为参数传给别的方法，那么这就是方法逃逸；如果没有方法逃逸，那不会被别的方法访问到这个对象，我们就可以在栈上分配这个对象；
  2. 如果对象定义后可能被别的线程访问，那这就是线程逃逸。如果没有线程逃逸，那么我们就可以消除同步，也就是不加锁。
