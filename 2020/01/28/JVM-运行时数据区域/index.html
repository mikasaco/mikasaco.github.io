JVM-运行时数据区域
发表于 2020-01-28| 更新于 2020-02-01|JVM运行时数据区域

运行时数据区域

方法区

  • 类加载之后的方法代码(字节码指令)、常量、静态变量、class 对象、即时编译之后的代码、类信息(类的方法名，返回参数，访问权限)等，都会存放在方法区；

  • 方法区是线程共享的；

运行时常量池

  • 运行时常量池是方法区中的一部分；

  • 运行时常量池中存放的是 class 文件的常量池(class 文件中的常量池也被称为静态常量池)，并且运行时常量池会对重复的字符串过滤。

运行时常量池：在方法区中，存的就是 class 文件的静态常量池的内容

class 文件中的静态常量池：class 文件中就有的

字符串常量池：在堆中，存的就是已有的字符串，做缓存

堆

  • 堆是 JVM 最主要的存放数据的区域，创建的对象一般都分配在堆上，垃圾回收也主要发生在堆上(方法区中也会发生垃圾回收)；
  • 堆是线程共享的；
  • 堆空间分为新生代和老年代，新生代又分为 Eden 区和两个 Survivor 区。

栈

  • 栈包括虚拟机栈和本地方法栈(主要一般讲的是虚拟机栈)；
  • 每个线程一个栈，每个栈中可以有多个栈帧(每个方法就是一个栈帧)，每个栈帧又包括局部变量表、操作数栈、方法返回地址等；
  • 局部变量表中存放方法参数和方法中定义的局部变量，局部变量表的大小在class 文件中就确定了；
  • Java 是基于栈的字节码解释执行引擎，其中的栈就是栈帧中的操作数栈。一个一个字节码指令操作的就是操作数栈，栈的大小也是在 class 文件中就确定了；
  • 栈是线程私有的。

程序计数器

  • 程序计数器的作用就是记录每个线程执行到了字节码的哪行，这里的字节码代码就是类加载之后存放在方法区的字节码指令；
  • 程序计数器是线程私有的。

直接内存

直接内存并不是 Java 虚拟机运行时数据区域的一部分，因为对象分配、方法执行等不是在直接内存上的，但是 Java NIO 提供了一种直接使用 Native 函数库分配堆外内存，然后在堆内创建一个 DirectByteBuffer 的引用的方式，可以减少数据拷贝次数。

Java8中虚拟机的改动

进入到 Java8 版本，虚拟机的内存布局也发生了重大的改变—移除了方法区，改用元空间。其实这句话是有问题，方法区并未移除，只是方法区的实现方式发生了改变。

在 Java8 之前虚拟机方法区的实现，在 Hotspot 中就是用永久代实现的，而 Hotspot 最流行，所以之前都把永久代等同于了方法区，但其实永久代只是方法区的一种实现方式。

为了解决方法区出现 OOM 的问题，Java8 中该用元空间来实现方法区。

对象的内存布局

  1. 对象头

    对象头中包括两部分信息：

    • Mark Word。包括锁标志(无锁、偏向锁、轻量级锁、重量级锁)、是否是偏向锁、持有锁的线程 ID、GC 年龄等；
    • 类型指针。类型指针指向了这个实例对应的类是哪个。

  2. 实例数据

    实例数据就是对象中真正存储的数据类型。

  3. 对齐填充

    对齐填充没有实际意义，虚拟机中对象起始地址必须是8的整数倍，而实例数据多少不确定可能不满足这个规律了，所以用对齐填充来补齐。

对象创建

当遇到 new 这个字节码指令时，就需要创建对象，步骤如下：

  1. 先会检查运行时常量池中 new 的参数对应的类的符号引用，然后检查这个类是否被加载了，如果没有加载，那就需要走类加载机制，把这个类加载进来；

  2. 当完成类加载之后，就可以在堆上为对象分配内存了，一般对象创建是分配在新生代 Eden 区，但如果是大对象也会直接分配在老年代。那具体如何分配，分配的算法有下面两种：

    • 新生代分配内存的方式是指针碰撞，也就是内存中已经分配的空间和未分配的空间放在两边，分配新对象内存的时候，只要移动指针就可以了，这种方式适合新生代采用复制算法，没有内存碎片。

    • 老年代如果采用标记-清除算法(CMS 就是标记-清除算法，且是老年代的垃圾收集器)，那会有内存碎片，所以也就用不了指针碰撞的方式分配内存，而是采用空闲列表的方式去分配内存。空闲列表就是一个列表记录了哪些内存还没用，然后分配的时候从列表中取；

    此外，堆线程共享，所以分配内存也要保证同步，JVM 是采用 CAS 和失败重试的方式保证同步；如果采用了本地线程分配缓冲，也就是堆中每个线程有单独的内存空间，分配内存的时候就在自己线程的内存区域分配，这样避免同步性能损耗，当然如果，本地线程分配缓冲不够用了，那还是需要 CAS 和失败重试的方式来分配；

  3. 内存分配完成之后，会把对象实例数据设置“零值”；

    类变量(类中 static 修饰的变量)在类加载的准备阶段会设置“零值”；

    实例变量(类中没有 static 修饰的变量)在对象分配，内存分配好之后会设置“零值”；

    方法中的局部变量是在栈帧的局部变量表上的，不会设置“零值”，所以如果不赋值不能使用。

  4. 为这个对象的对象头，设置数据；

  5. 虚拟机角度上面的步骤以及完成了对象的创建了，但是用户其实还需要在去调用实例构造器 < init > 的方法，来完成对实例数据的设置。一般 new 字节码指令后面还会执行 < init > 方法。

对象的访问定位

我们知道在方法执行时如果要用到某个对象，栈帧中的局部变量表中会持有这个对象的引用，对应的数据类型是 reference，通过这个引用我们就可以去访问到对象，进而获取/修改对象的实例数据。那怎么去访问定位这个对象呢？一般有两种算法：

  1. 句柄访问

    句柄访问会在堆中开辟一个句柄池，栈上的局部变量表中引用存的地址是句柄池中指针的地址，然后句柄池中再持有对象的地址。

    优点：对象发生改变(例如 GC 后调整了位置)不需要改变栈上的引用数据；

    缺点：需要多一次定位访问。

  2. 直接访问

    直接访问就是栈帧上局部变量表上存的就是堆上对象的起始地址。

    优点：少了一次访问定位；

    缺点：对象发生改变，栈上的数据也需要改变。

程序执行的流程

  1. Java 文件通过 javac 编译器编译成 class 文件；
  2. class 文件通过类加载机制加载到虚拟机的方法区中；
  3. 如果我们需要执行加载的类的方法，那么就会在栈中对应的线程上创建一个栈帧(程序入口 main() 方法就会在把这个类加载之后，在栈空间的 mian 线程栈上创建一个栈帧)；而如果执行的代码需要创建一个加载的类的实例对象的话，就会在堆上分配内存
